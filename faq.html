FAQ


General

How much faster is Caffeine than HTTP/S

For most requests, fully encrypted Caffeine is ~3x-8x faster than HTTPS and 2.5-5x faster than HTTP.

Why can't I download your SDK from the site?

Caffeine is currently in a closed beta. If you contact us we should be able to get you up and running.

Does Caffeine work for Android and the web as well?

Caffeine's Android release is on our product roadmap. If you would like us to expedite that, please let us know at team@caffei.net. There are no plans for web, because we believe the future is mobile.

Can I see a demo?

To view a live demo, set up a time for a demo here.

How secure is Caffeine?

All all network requests are fully encrypted to prevent man in the middle and DDoS attacks. Security is not a feature, but philosophy for us.

Do you have access to our data?

We care deeply about the privacy of business' data. As a company, we do not have access to your data.

Technical

How does Caffeine work?

We speed up client-server communications by stripping out bloat from the web. Caffeine is a vertically integrated networking stack from the client side networking library to TCP. We were able to make hundreds of optimizations across the networking stack, that HTTP(S) could not, because we know how requests will be handled on the server and the client.

Do I have to rewrite my app?

No. It is as simple as dragging an dropping a file and adding one line of code to the app to get started. It hooks into your existing client side networking library (AFNetworking, CFNetworking, Alamofire, etc.) and proxies traffic to your existing web werver.

Will hybrid apps or webviews get sped up?

Currently we will only speed up the native iOS portions (written in Swift or Obj-C). Caffeine for webviews is coming soon.

What if something goes wrong with Caffeine?

Caffeine was designed to preserve the user's session at all costs. Our failsafe defaults to your your legacy stack (HTTP/) if we are facing any issues. Your users will not be interupted.

What software does Caffeine integrate with?

Caffeine is just like any other networking communication——it operates in the backround. It doesn't matter what language your backend is written in or what web server you are using.

Do I have to use Caffeine as a proxy?

Depending on your desired server configuration you can install Caffeine using a docker image server side. This removes additional latency, and you will notice bandwidth savings as Caffeine will not acts as a middle man.

Are you using UDP?

No. Caffeine is currently a TCP based implimentation.

Will all apps see the same speed improvement?

If your app is already fast we will probably make it faster. Caffeine's relative speed improvements over HTTP/S maintain for all network requests under 5mb, after that speed improvements begin to taper off.
